# **🧩** 

# **Level 0：真值表（Truth Table）**





**输入 → 输出的完整枚举**

| **A** | **B** | **AND** |
| ----- | ----- | ------- |
| 0     | 1     | 0       |

此时你拥有的东西是：

```
逻辑是可枚举、可计算、可预测的。
```

这一步形成**确定性**与**可验证性**的基础。



🔜 下一步 → 真值表组合。



------





# **🧩** 

# **Level 1：布尔逻辑（Boolean Logic）**





真值表可以组合，产生 **更复杂的逻辑门**：

```
NAND → 所有逻辑门
OR → 分支
XOR → 差异与加法
NOT → 条件反转
```

此时你得到一个核心事实：



> 任何逻辑都能由有限真值表组合构成。



🔜 下一步 → 让逻辑门做“运算”。



------





# **🧩** 

# **Level 2：加法器（Half-Adder / Full-Adder）**





真值表组合 XOR + AND → **半加器**



| A | B | SUM(XOR) | CARRY(AND) |

|—|—|—|

| 1 | 1 | 0 | 1 |



两个半加器组合 → **全加器**

多个全加器串联 → **二进制加法器（Ripple Carry Adder）**



第一次出现奇迹：



> 💥 逻辑 → 能做算术了

> （计算的火花第一次点燃）



🔜 下一步 → 加法 + 状态记忆。



------





# **🧩** 

# **Level 3：寄存器 & ALU**





加法器会算，但是没有记忆。



加入 **触发器（Flip-Flop）** → 可以存 0/1

加入 **时钟（Clock）** → 数据按节奏流动

加入 **选择器/多路复用器（MUX）** → 选择操作



于是形成 **ALU（算术逻辑单元）**

```
ALU = XOR + AND + OR + ADDER + NOT + SHIFT
```

> 真值表 = 运算

> 触发器 = 记忆

> ALU = 能运行的数学机器



🔜 下一步 → 让机器执行序列动作，而不是一次计算。



------





# **🧩** 

# **Level 4：指令集（ISA）**





我们给电路命令：

```
ADD R1, R2, R3     # R3 = R1 + R2
LOAD R1, [0xFF]
JUMP IF ZERO 0x90
```

这是机器语言。



CPU 通过控制单元（Control Unit）将指令翻译成：

```
→ 使用哪个加法器
→ 哪些寄存器输出
→ 何时写回
→ 是否跳转
```

**所有指令最终都由真值表流水组合驱动完成。**



🔜 下一步 → 可运行程序。



------





# **🧩** 

# **Level 5：CPU → 汇编 → 操作系统**





真值表 → 逻辑门 → 加法器 → 寄存器 → CPU → 汇编 → OS



内存管理、文件系统、进程调度、中断、系统调用……



全由低层逻辑组合而来。



> 操作系统 ≈ 最高规模的真值表组合

> CPU ≈ 真值表的持续执行器



🔜 下一步 → 高级语言。



------





# **🧩** 

# **Level 6：编译器（Compiler）**





C语言一行代码：

```
x = x + 1;
```

会被编译为：

```
LOAD x → R1
ADD R1, 1 → R1     # XOR+AND 构建的全加器
STORE R1 → x
```

最终还是执行真值表逻辑门。



🔜 下一步 → AI / 高维抽象。



------
